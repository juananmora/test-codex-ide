Core Concepts

What each relationship means

These relationships are easier to reason about when grouped by responsibility instead of by folder name alone.

1. Entry points define the frame

pages/_app.tsx and pages/_document.tsx are not normal UI sections. They control the app wrapper and base document, so other files sit inside their structure.

2. The page file is the orchestrator

pages/index.tsx does not render everything by hand. It imports focused components and assembles them in order, which keeps the route readable.

3. Components depend downward

Higher-level sections like ExamplesSection depend on smaller pieces such as Section, CodeExample, and icon components. The dependency direction moves from broad layout to small reusable pieces.

4. Styles are cross-cutting

styles/globals.css is imported once in pages/_app.tsx, but its rules affect every page and component. That makes it a shared layer, not a local dependency.

5. Public files are referenced, not imported

Files in public/ are served statically. Components and document metadata refer to them by URL, such as /favicon-light.png or logo paths.

6. Data can join the graph too

The relationship is not only visual. In pages/index.tsx, data fetched in getStaticProps() flows into ExamplesSection and then into the repo card list.

Concrete Examples

Actual file-to-file links in this repository

These are specific relationships confirmed from the current codebase, not placeholders.

pages/_app.tsx → styles/globals.css

The global stylesheet is imported once at the application level so every page shares the same base typography, theme variables, and utility behavior.

pages/_document.tsx → public/favicon-*.png

The custom document links the light and dark favicons. That makes the HTML document aware of static assets from public/.

pages/index.tsx → components/*

The landing page imports Hero, WhySection, CompatibilitySection, ExamplesSection, and more to compose the route.

components/ExamplesSection.tsx → ExampleListSection

The examples area is layered. A higher-level section wraps layout and then delegates repo cards to a more focused list component.
